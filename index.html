How to rank features and define metrics — a methodology survey

A field study of how 20 of the most-studied software companies actually rank features and decide which metrics matter. The point is to compare the published methods side-by-side so the team can pick (or mix) the one that suits us.

Goal of this research

Find a suitable method — to rank features and define metrics — that we can apply across the whole company.

We're surveying what other successful product companies actually do (with sources), so we can have a grounded discussion about which approach (or which combination) to adopt. This is research material to read and discuss, not a recommendation.

Scoring Numeric ranking formulas

Produce a number to rank ideas. Cheap to run, easy to game; best as a tool to expose disagreements about Reach/Impact/Confidence.

RICE — Reach × Impact × Confidence ÷ Effort
From: Intercom
The most-cited scoring formula. Fixed scales, transparent math, easy to audit.

Input Modeling Map work to a single output metric

Build a tree from a "North Star" outcome down to the inputs teams can move. Helps every team see whether their work moves the company-level metric or not.

North Star metric & input model
From: Amplitude, Slack, Atlassian
One number for the whole company; inputs cascade down to team-level OKRs.

Experimentation Measure, don't estimate

Ship a controlled test and let the data decide. The only category that produces a causal answer; pays for that rigour in infrastructure cost and slower decisions.

ExP · OEC · CUPED
From: Microsoft (Bing, Office, Azure)
The classic platform — single Overall Evaluation Criterion + variance reduction.
LiX → XLNT → T-REX
From: LinkedIn
Same discipline as Microsoft, told as a platform-investment story.
ABlaze + Causal Inference
From: Netflix
A/B for what's randomizable; quasi-experiments for what isn't.
Booking's democratized experimentation
From: Booking.com
~25,000 tests/year. Hypothesis-first, SRM-checked, standard metric library.
ERF — Experimentation Reporting Framework
From: Airbnb
Shared metric DSL + standard reports + variance reduction.
XP + universal holdout
From: Uber
Staged ramp 1→5→25→50→100% + quarterly cumulative-effect measurement.
Switchback experiments
From: DoorDash
For marketplace SUTVA violations — randomize region × time-window cells.
A/B + time-split + bandits
From: Lyft
Three designs for three kinds of marketplace problem.
Pinterest's A/B platform
From: Pinterest
Lightweight config + cross-platform consistency + real-time monitoring.

Goal-laddering / Define-first Pick the metric before the work

Force the team to name what success looks like — and how it will be measured — before committing to build. Cheap discipline, high payoff.

HEART + Goals-Signals-Metrics
From: Google (CHI 2010)
5 user-experience dimensions, each laddered down to a specific metric.
DIBB — Data → Insight → Belief → Bet
From: Spotify
Every bet must trace back through evidence; antidote to "we just feel like it."
V2MOM — Vision · Values · Methods · Obstacles · Measures
From: Salesforce (Benioff, 1999)
Top-down alignment cascade; every team's plan rolls up to the company V2MOM.
Pyramid of Clarity
From: Asana
Mission → Strategy → Objectives → Key Results → Projects.

Written Discipline Make the team write the bet before they make it

No formula, no number — just a structured document peer reviewers can engage with. Works only if reading and pushing back is taken seriously.

PR-FAQ · Working Backwards
From: Amazon
Write the launch press release + customer FAQ before you build. If it's not compelling, don't build.
Shaping + deep-dives + long-term filter
From: Stripe
No scoring formula. Qualitative judgement reinforced by writing culture and a long-term direction.
