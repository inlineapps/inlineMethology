Pyramid of Clarity — Asana's mission-to-tasks ladder

Five layers, top to bottom: Mission → Strategy → Objectives → Key Results → Projects (with tasks underneath). Asana built it because they found classical OKRs hard to apply consistently — the Pyramid is their simpler version that still ladders every task to the mission.

Goal

The honest headline: alignment by traceability, not by scoring

The Pyramid of Clarity answers one question: "Why are we doing this task?" If you can trace any task upward through Projects → Key Results → Objectives → Strategy → Mission, the work is justified. If the trace breaks anywhere, it's a flag to either reshape the task or — more honestly — drop it.

Like Salesforce's V2MOM, it is not a feature-impact estimator. It tells you whether work aligns, not whether it will succeed. Asana's own claim: it "creates clear and complete alignment so you can make better decisions and prioritize work that drives measurable impact" — alignment plus measurable KRs is the priority signal.

The five layers, top to bottom

The shape is literal — wider at the bottom because many tasks roll up to fewer Projects, fewer KRs, fewer Objectives, one Strategy, one Mission. Every layer is supposed to reference the layer above.

MissionWhy the company exists. Decade-scale.

StrategyHow we'll get to the mission. Multi-year.

ObjectivesWhat we want to achieve this period (1–4 quarters).

Key ResultsMeasurable outcomes that say the Objective happened.

Projects (then Tasks)The actual work that moves the KRs.

Asana's framing: "Complete a task, you make progress on a project. Complete the project, you advance a goal. Advance enough goals, you move toward the mission."

Source: Asana — Pyramid of Clarity (Dustin Moskovitz) · Wavelength — Pyramid of Clarity.

How the Pyramid actually picks work

The mechanism is upward traceability + measurable KRs. Asana built their product so the linkage is enforced — every Project in Asana can be attached to a Goal, which is attached to a parent Goal. The model assumes the structure is visible.

1

The Mission is set and rarely changes

Asana's own Mission: "Help humanity thrive by enabling all teams to work together effortlessly." Decade-scale. Provides the constant North that the rest of the pyramid resolves against.

2

Strategy translates Mission into a few multi-year bets

This is the layer V2MOM's Methods live in. Asana folds it under "Strategy" — same idea, different label.

3

Objectives + Key Results — the OKR layer

Each period (Asana uses quarters), every team writes Objectives that advance one Strategy, with Key Results that are measurable, time-bound, ambitious. This is the only quantitative layer.

4

Projects must be tied to a KR

The forcing function: in the tool, a Project without a parent Goal is visibly unattached. Teams must either tie it to a KR or justify why it's still on the list.

5

Tasks roll up automatically

Daily-level work inherits its justification from the Project it sits in. The promise: any individual can trace why their task matters by walking the pyramid upward.

Pyramid vs V2MOM vs classical OKRs — what's the difference?

These three sit in the same category (alignment, not scoring) and overlap a lot. Quick map so the meeting can pick deliberately.

Pyramid of Clarity (Asana)

  • Five layers: Mission · Strategy · Objectives · Key Results · Projects.
  • Enforced as a tool in Asana — every Project has a parent.
  • Adopts OKR as the measurable middle layer; lighter "hierarchy" than V2MOM.

V2MOM (Salesforce)

  • Five sections per person/team: Vision · Values · Methods · Obstacles · Measures.
  • Each person writes one; cascade up to CEO.
  • Includes Values and Obstacles explicitly — Pyramid does not.

OKRs alone (Doerr / Google) are just Objectives + Key Results, no Mission/Strategy/Values wrapper. The Pyramid and V2MOM both wrap OKRs in additional structure — Pyramid via mandatory parent layers, V2MOM via Values + Obstacles. Choose by what gap your org actually has.

What the Pyramid does not do

It does not score features. A Project tied to a KR is "justified," not "high-impact." If three Projects are all tied to the same KR, the Pyramid doesn't tell you which to do first. That's where RICE, the North Star Framework, or experimentation come in.

It is only as honest as its KRs. If KRs are vanity metrics (output counts, "ship X features"), the Pyramid laddering will look perfect while the company is going sideways. The discipline is forcing outcome KRs, not output KRs.
Why the Pyramid is on the list anyway. It is the most concrete artifact of "every task must explain itself." That alone catches the common planning failure mode — items that are on the roadmap because they were on last sprint's roadmap, never asked which outcome they're meant to move.
